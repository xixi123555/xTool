# 执行流程（必须严格按步骤执行）

1. **接收用户需求**
   理解用户要实现的功能、任务、重构或开发目标。

2. **生成结构化计划**
   输出内容必须包含：
   - 任务目标
   - 涉及文件清单
   - 执行步骤（按顺序）
   - 风险与注意事项
   - 验收标准
   及其他计划的内容

3. **创建目录（如不存在）**
   自动创建目录：
   ```bash
   .cursor/plans/

4. **计划命名规则**
    计划文件名格式：`YYYYMMDD-任务名-plan.md`（例如 `20260327-xtoolMCP_HTTP-plan.md`）

5. **文件排序规则**
    生成的文件按照时间排序，最新生成的排在最上面
